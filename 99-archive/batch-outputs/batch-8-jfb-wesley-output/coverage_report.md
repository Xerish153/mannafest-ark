## Idempotency

Both extractors are deterministic — rerunning produces identical JSON.
The `build_*_sql.py` scripts emit identical filenames + row orderings on
rerun. `run_all.py` queries `commentaries` before each apply and skips
already-present `(scholar_id, book_id, chapter)` combinations, so full
or partial re-runs are safe.

## Source format notes

- **JFB**: `/MannaFest/jfb-data/JFB{01..66}.htm` + 6 intro files (JFB00*).
  Standard CCEL Public-Domain HTML. `<A NAME="ChapterN">` anchors for
  most books; single-chapter books (2 John / 3 John / Jude) use
  `<A NAME="Introduction">` + verse anchors instead — handled via
  fallback in `extract_jfb.py`.

- **Wesley**: `/MannaFest/wesley-data/notes.doc` — MS Word saved-as-HTML
  with ThML-derived CSS classes (`div1`..`div5`). File IS valid HTML
  despite the `.doc` extension (first bytes: `<html xmlns:v="urn:schemas-
  microsoft-com:vml"`). Parsed with bs4 using `class=div2` for book
  boundaries and `class=div3` for chapters (Roman numerals for most
  books, arabic numerals for Psalms, `Commentary on {Book}` fallback
  for single-chapter Obadiah).
