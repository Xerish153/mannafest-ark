### Phase 3 — Extraction pipeline

**`extract_calvin.py`** (110 lines Python, no deps beyond stdlib). Key design decisions:

1. **Verse-range header as chapter anchor**, not `CHAPTER N.` markers. CCEL's chapter markers vary across volumes (`CHAPTER 1.` vs `CHAPTER 1` vs `PSALM 1.` vs none-at-all for the first chapter of each volume) and are sometimes missing (Calvin merged Gen 32+33 and 34+35 without inserting visible `CHAPTER` markers). Verse-range headers like `Genesis 1:1-31`, `Hebrews Chapter 3:1-6`, `Jude 1-2` (verse-only for single-chapter books), `Psalm 1:1-2` are universally present.

2. **Best-candidate selection for chapter boundaries.** Preface citations like `Genesis 1:1-6` (Leo Juda translator comparison in calcom01 front-matter) collide with the real `Genesis 1:1-31` chapter header. Heuristic: among candidates with `vstart=1`, pick the one with the largest `vend` (chapter headers cover the full first listing block; prefaces cite snippets).

3. **Commentary-prose detection via paragraph-pair lookahead.** The verse-listing block has English + Latin pairs, both starting with the same verse number. The commentary block has standalone prose paragraphs. Heuristic: find the first paragraph ≥4 lines starting with `N.` or `N` where the NEXT paragraph does NOT also start with `N.` — that breaks the English/Latin pairing and marks the commentary entry.

4. **Single-chapter book support** for Philemon, Jude, Obadiah, 2 John, 3 John: separate regex `^\s*(Book) \s+\d+(-\d+)?$` matching verse-only ranges. Obadiah gets chapter=1 implicitly.

5. **Light OCR cleanup.** Strip `[NN]` footnote markers. Normalize whitespace. Collapse intra-paragraph newlines. No paraphrasing, no summarization — verbatim PD source text.

6. **Truncation at 500 words** per §3.3 spec (body target 200-500 words per chapter-level row). Cut at sentence boundary if possible; else word boundary.

**`build_calvin_sql.py`** (JSON → SQL). First-occurrence-wins dedupe across volumes (handles calcom22/23 Ezekiel overlap and calcom11/12 Psalm 119 overlap). Filters impossible chapters (1 rejection: Mark 21, a CCEL typo — Mark has only 16 chapters). Emits 10-row INSERT batches sized to fit Supabase MCP's `execute_sql` call budget.
