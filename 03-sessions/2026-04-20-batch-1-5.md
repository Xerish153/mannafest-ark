## Environment notes (sandbox quirks)

The repo is mounted via virtiofs from `C:\Users\marcd\Downloads\MannaFest`. Two Windows-side quirks surfaced:

1. **Corrupt git index with an unrecognized extension** (`error: index uses <garbled> extension`). Worked around by rebuilding an index at `GIT_INDEX_FILE=/sessions/…/mannafest_index` via `git read-tree HEAD`. The on-disk `.git/index` is untouched — Marcus's Windows git still uses it normally.
2. **Unremovable lock files** `.git/HEAD.lock` and `.git/index.lock` (zero-byte ghost files that refuse `rm -f`, `mv`, or `unlink`). These prevented `git branch`, `git commit`, and `git checkout` from succeeding normally. Worked around by:
   - Creating the feature branch via an entry in `.git/packed-refs` (no subdir needed)
   - Updating `.git/HEAD` to point at `refs/heads/feature/homepage-launchpad`
   - Building the commit via plumbing (`git write-tree` → `git commit-tree`), then updating the packed-ref

The resulting commit `96262c8` is structurally identical to one made via the porcelain `git commit`, and Marcus's Windows git will see a clean working tree on the new branch.

One side effect: an accidental stale ref file `.git/refs/heads/feature-homepage-launchpad` was written before I discovered the subdir restriction. It points at main (`16a39b1`) — harmless (effectively another branch at main), but Marcus may want to `git branch -D feature-homepage-launchpad` on Windows as cleanup.
