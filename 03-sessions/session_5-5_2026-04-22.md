## Known quirks / non-blockers surfaced

- **Duplicate graph_edges rows.** Genesis 15:1 appears 3x in Romans
  8:31's outgoing edges (and similar patterns for other top-ranked
  targets). Handled in `CrossReferenceSection` + `OverrideEditor`
  via render-time dedup by target verse. A proper DB-level collapse is
  out of scope here; flag for a future hygiene batch.
- **All graph_edges.weight = 1.0** across the cross_reference relationship
  type. The backfill ordered by weight DESC, target_node_id ASC; since
  weight is uniform, the effective ranking is target_node_id ASC. This
  is the prompt's documented fallback ("If no edge weight: display_rank
  = ROW_NUMBER() ... ORDER BY target_book, target_chapter, target_verse")
  in spirit. The top-5 for Romans 8:31 surfaces Genesis 15:1 at #1,
  which is a defensible pairing ("Fear not... I am thy shield" vs. "If
  God be for us, who can be against us") — enough signal for v1.
  Richer ranking can arrive via a future Guzik-tier-weight or TSK-tier
  ingest batch without touching this library code.
- **MCP apply_migration timeout at 60s** on migration 050. The UPDATE
  did commit — post-check query confirms all 1.3M rows ranked. Flagging
  so Marcus knows the apparent timeout is a false negative.
