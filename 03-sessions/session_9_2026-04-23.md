# Session 9 — Jesus Multi-Catalog (Cowork)

## Scope decision — recorded here because it drove everything

Spec asked for ~127 pages with `body_markdown` "compiled from PD commentary." That violated OPERATING_RULES §3 without reading the ingested PD corpus (in Supabase, not in sandbox) per passage. Marcus locked A/A/A:

1. **Structural-only ship**, draft status, Pastor Marc authors body post-ship.
2. **Shared admin editor** at `/admin/jesus/[cluster]/[slug]` for all 7 clusters.
3. **35 miracles** including `boy-with-demonic-seizures` as Exorcism #5 (same event as `epileptic-boy` healing; catalogued twice under its deliverance aspect).

And three additions:

1. **Consume 8.9 pull_quotes on cluster pages** — `<PullQuoteList />` scoped to each page's parsed `primary_verse_ref`. Zero per-page authoring; the moment Pastor Marc curates in `/admin/commentary/:book/:chapter/verses`, the cluster page lights up.
2. **Publish-guard at the write boundary** — the `/api/admin/jesus/[cluster]/[slug]/publish` API route refuses the `draft → active` flip if refCount < 3. Returns HTTP 409 with a message the admin editor surfaces.
3. **Seed summary fields only, never body fields** — `body_markdown`, `spiritual_significance`, `old_testament_type`, `editor_note` seeded NULL across all 119 members. Only factual fields (slug, title, subtitle, cluster_group, primary_verse_ref, synoptic_parallels, is_messianic_sign, one-sentence factual summary, scripture refs) land at seed time.

## Delivered

### Migrations (7)

`063_jesus_miracles.sql` → `069_jesus_apostolic_figures.sql`. Each mirrors 7-B shape:

- `BIGSERIAL` primary key, `book_id BIGINT REFERENCES books(id)` on the refs table (NOT the `UUID` + `book_slug` in spec's example SQL — pattern-wins).
- `status` gate on public read RLS (`status = 'active'`); `no_direct_write` lockdown; set_updated_at trigger; indexes on `(status, display_order)` + `(book_id, chapter, verse_start)` for the featured-on-verse join.
- Cluster-specific columns: `is_messianic_sign` (miracles), `parable_setting` + `jesus_interpretation` (parables), `outline_structure` JSONB + `key_themes` + `central_claim` + `gospel_source` (discourses), `day_label` + `day_ordinal` + `timeline_events` JSONB (passion week), `appearance_ordinal` + `participants` + `location` (resurrection appearances), `birth_name` + `is_judas` + `call_narrative_ref` + `death_tradition` + `primary_book_slug` (apostles).
- Sevens gets an extra table `jesus_seven_items` (1-to-7 items per catalog page) — that's the "7 items with their own subsection" shape the spec called for.
- Apostles refs add two new `ref_type` values: `call` (call narrative) and `authored_book` (books attributed to the apostle).

## Decisions made per autonomy clause

1. **Schema shape = 7-B, not spec example.** Spec SQL used `UUID` + `book_slug TEXT`. Established pattern (057, 058) is `BIGSERIAL` + `book_id BIGINT REFERENCES books(id)`. Pattern wins per mirror-7-B directive.
2. **featured_page_refs column names** — spec example called them `slug` + `title`; actual schema (052) is `featured_page_slug` + `featured_page_title`. Went with actual schema.
3. **Miracle #8/#35 overlap** — kept Canaanite daughter under Healings, shipped 35 miracles with `boy-with-demonic-seizures` as Exorcism #5 (same event as `epileptic-boy` healing, catalogued twice by its deliverance aspect). Subtitle on the exorcism page cross-references the healing.
4. **Sevens table design** — added `jesus_seven_items` (1-to-7 child table) instead of a JSONB `items` blob. Cleaner RLS, cleaner index, cross_ref_slug + cross_ref_route_prefix support inter-cluster linking at the item level.
5. **Discourse refs** — each discourse seeds its chapter range as multiple `primary` refs (one per chapter) rather than one wide range. This makes the featured-on-verse join trivial — any verse in Matthew 5–7 pulls the Sermon page.
6. **Apostles ref_type extensions** — added `call` + `authored_book` to the refs enum on the apostolic-figures refs table only. Other clusters keep the original 5-value enum.
7. **Editor attribution on Judas** — `is_judas=TRUE` is a flag; the actual "multiple PD voices, no resolution sentence" discipline is enforced by Pastor Marc via the editor note, not seeded prose. The admin editor placeholder label reminds him.
